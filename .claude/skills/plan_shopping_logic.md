# Skill: Plan Shopping Logic & Architecture

## Description
Prevents unauthorized architectural changes and ensures a clear plan exists before implementing complex shopping filters or multi-mall price comparisons.

## Trigger
- User asks for a complex new feature (e.g., "Filter only for N-Pay items").

## Instructions
1. **Analyze Requirements:** Understand the specific shopping query needs.
2. **Create Plan:** Document the approach in `docs/plans/`.
3. **Define Output Format:** Specify how the agent should present the result (e.g., Table vs. List).
4. **Wait for Approval:** Present a summary to the user and wait for "Go" before coding.
